[안드로이드  앱 설계서 ] 도트노트  (DotNote)  
2022145053 스마트콘텐츠학과  윤석천  
1. 앱의 이름 
도트노트  (DotNote)  
• 부제: AI 기반 지식 구조화  및 시각화  메모 도구 
2. 앱의 개발 동기 
• 정보 과부하의  시대: 매일 쏟아지는  파편화된  아이디어를  단순 리스트로  관리하면  
정보 간의 유기적인  맥락을  놓치기  쉬움. 
• 비선형적  사고의  한계: 인간의  사고는  그물망처럼  연결되어  있으나 , 기존 메모 앱
은 선형적 (List) 구조에  갇혀 있어 창의적  발상을  저해함 . 
• AI 기술의  모바일  접목: 최신 LLM(Gemini) 과 안드로이드의  모빌리티 (위치, 음성, 
카메라 )를 결합하여 , '적기만  하면 알아서  정리되는 ' 지능형  메모 경험을  제공하고
자 함. 
• 단순한  메모 기록을  넘어, 지식 관리 방법론인  **제텔카스텐 (Zettelkasten)** 을 모
바일 환경에서  자동화하고자  함. 사용자가  메모를  분류하기  위해 들이는  '분류 스
트레스 '를 제거하여  **인지적  부하(Cognitive Load)** 를 최소화하고 , 본질적인  아이
디어 발상에만  집중할  수 있는 환경을  제공하는  것이 핵심. 
3. 기존의  유사한  앱 분석 
• Obsidian / Roam Research : 메모 간 양방향  링크와  그래프  뷰를 제공하지만 , PC 
중심의  무거운  인터페이스와  마크다운  문법 숙련도가  필요함 . 
• Xmind / GitMind : 마인드맵  생성에  특화되어  있으나 , 일상적인  짧은 메모를  자동
으로 분류하거나  메모 간의 복잡한  네트워크  관계를  표현하는  데 한계가  있음. 
• Google Keep / Evernote : 입력은  간편하지만 , 수천 개의 메모가  쌓였을  때 정보 
간의 연관성을  시각적으로  파악하기  어려움 . 
4. 자신의  앱과 기존의  앱의 차별점  분석 
• AI 자동 구조화  (Automation) : 사용자가  직접 링크를  걸거나  폴더를  지정할  필요 
없이, AI가 텍스트 /이미지 /음성을  분석하여  최적의  위치(노드)에 배치함 . 
• 하이브리드  시각화  (Dual-View) : 엄격한  분류가  필요할  때는 '마인드맵 ', 자유로운  
발산이  필요할  때는 '네트워크  그래프 '로 즉시 전환 가능. 
• 시계열  및 공간 정보 결합 (Time & Space) : '타임라인  슬라이더 '를 통한 생각의  
변천사  추적 및 지도(Map) 기반의  위치별  메모 탐색 기능을  제공하여  모바일  특
화 경험 극대화 . 
• 멀티모달  퀵 캡처 (Multi -modal Quick Capture)  
일상에서는  텍스트를  입력하기  어려운  상황(운전 중, 요리중  등)이 많은데 , 이러한  
상황에서도  음성 인식을  통하여  AI가 즉시 텍스트로  변환(STT)하고, 메모에  즉시 
반영. 
• Privacy -First & Low Latency: 대다수의  AI 메모 앱이 클라우드  의존적인  것과 달
리, 본 앱은 On-device ML Kit 를 적극 활용하여  개인정보  유출 우려를  불식시키고  
오프라인  상태에서도  끊김 없는 저지연 (Low Latency) 반응 속도를  보장. 
 
5. 앱의 화면 구성 (Wireframe Concepts)  
• 메인 피드 (Main Input) : 하단바  중심의  퀵 버튼(텍스트 , 음성, 카메라 )과 실시간  
AI 분석 상태창 . 
• 구조화  뷰 (Visual Canvas) : 전체 화면을  활용한  줌인/아웃 가능 캔버스 . 상단 스
위치로  마인드맵 /네트워크  모드 전환. 
• 타임라인  슬라이더  (Timeline Bar) : 화면 하단에  위치하여  특정 기간의  메모만  필
터링하여  캔버스에  노출. 
• 상세 메모 카드 (Detail Card) : 노드 클릭 시 나타나는  슬라이드  업 패널. 본문과  
AI가 추천한  연관 메모 목록 표시. 
• 디자인  시스템 : Google 의 최신 디자인  시스템인  **Material 3(M3)** 를 적용하여  사
용자 설정에  따른 동적 색상(Dynamic Color) 을 지원하고 , 일관성  있는 UX를 제공. 
• 제스처  인터페이스 : Canvas 기반의  시각화  뷰에서는  핀치 투 줌(Pinch -to-zoom)  
및 멀티 터치 드래그 를 지원하여  방대한  네트워크  지도에서도  직관적인  탐색이  
가능하도록  설계. 
 
 
 
 
6. 앱의 기능 
1. 스마트  입력 (Smart Capture)  
• 말하는  대로 메모 (STT): 운전 중이나  요리 중처럼  타이핑이  어려울  때 음성으
로 바로 기록하고  텍스트로  변환. 
• 찍으면  바로 텍스트  (OCR): 책이나  문서의  사진을  찍으면  AI가 글자를  인식해  
즉시 메모로  저장. 
• AI 자동 태그: 메모를  적기만  하면 AI가 문맥을  파악해  '시맨틱  태그'를 달고 
자동으로  주제를  분류. 
2. 하이브리드  시각화  (Hybrid View)  
• 생각의  지도 (Network Graph): 파편화된  메모들을  거미줄처럼  연결해  아이디
어 사이의  숨은 연관성을  한눈에  보여줌. 
• 논리 마인드맵  (Mind Map): 복잡한  정보들을  주제별  계층 구조로  깔끔하게  정
리. 
• 자유로운  시각 전환: 버튼 하나로  마인드맵과  네트워크  그래프를  오가며  생각
을 정리하거나  확장 가능. 
3. 맥락 기반 탐색 (Contextual Search)  
• 타임라인  슬라이더 : 화면 하단의  바를 조절하여  특정 날짜나  기간에  작성된  
메모만 찾기 가능. 
• 생각의  발자취  (GPS Map): 내가 어디서  이 생각을  했는지  지도 위에 표시해  
장소와  함께 기억을 상기. 
• 지능형  과거 알림 (Geofencing): 특정 장소에  가면 그곳에서  적었던  과거 메모
를 다시 보여주며  아이디어를  상기. 
 
7. 구현 우선순위  
1순위: 핵심 아키텍처  및 데이터  루프 (Must -have) 
목표: 메모 입력에서  AI 분석, 로컬 저장까지  이어지는  기본 파이프라인  완성 
• Gemini API 연동: 사용자  입력 텍스트에서  키워드 (태그)를 추출하고  시맨틱  임베
딩을 수행하는  백엔드  로직. 
• Room DB 설계: 메모 본문, 추출된  키워드 , 노드 간의 부모-자식 및 연관 관계를  
저장하는  그래프  구조형  테이블  구축. 
• 기본 UI: 텍스트  입력부와  저장된  메모 리스트  확인 기능. 
2순위: 하이브리드  시각화  엔진 (Critical -path) 
목표: Canvas 기반의  동적 레이아웃  구현 
• Canvas API 활용: 표준 뷰 계층을  벗어나  저수준  그래픽  제어를  통한 노드 및 선
(Edge) 렌더링 . 
• Force-directed 알고리즘 : 노드 간 반발력과  인력을  계산하여  최적의  네트워크  배
치를 자동 계산하는  물리 시뮬레이션  적용. 
• 마인드맵  레이아웃 : 중심 주제를  기준으로  계층적  트리 구조를  형성하는  배치 로
직 구현. 
3순위: 멀티모달  입력 및 시계열  필터링  (Should -have) 
목표: 모바일  기기 성능 활용 및 사용자  탐색 편의성  증대 
• On-device ML Kit: 카메라  촬영 시 실시간으로  텍스트를  인식(OCR)하여 즉시 메
모로 변환. 
• 타임라인  슬라이더 : 하단 바 형태의  UI를 통해 특정 기간 내 작성된  메모들만  캔
버스에  필터링하여  노출. 
• STT 연동: 음성 입력을  통한 텍스트  변환 및 자동 요약 기능. 
4순위: 지능형  서비스  및 공간 맥락 (Nice-to-have) 
• 유사도  기반 자동 연결: 새 메모 작성 시 벡터 유사도가  높은 과거 메모를  AI가 
감지하여  연결을  제안. 
• 생각의  지도 (Google Maps): 작성 시 저장된  GPS 정보를  활용하여  지도 위에 클
러스터링된  메모 뭉치 시각화 .
